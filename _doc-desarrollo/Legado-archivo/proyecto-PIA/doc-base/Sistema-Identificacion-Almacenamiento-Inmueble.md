# Sistema de identificación y almacenamiento por inmueble

---

La función de esta tabla es explicar cómo el sistema convierte cada inmueble en una unidad documental única y organizada. Presenta el Código Id de Inmueble como identificador central y muestra cómo ese identificador estructura carpetas, nombres de archivo y persistencia de resultados. Facilita entender la lógica interna de almacenamiento y recuperación.


| ID | Elemento                            | Qué es                                                                                                                             | Cómo se forma o nombra                                                                                                                                                   | Para qué sirve                                                                                            |
| -: | ----------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
|  1 | CII                                 | Es el **Código Id de Inmueble** asignado a cada proyecto.                                                                          | Se compone de **2 dígitos del año** y **dos del** mes en que se da de alta el proyecto + **4 dígitos numéricos** (rellenando con cetos izq) con el **ID del registro**  en  tabla proyectos.                                                                                                         | Identifica de forma única el inmueble dentro del proyecto y se guarda en la tabla del proyecto.           |
|  2 | Carpeta principal                   | Es la carpeta base del sistema de archivos donde se guardan los archivos de todos los proyectos.                                   | Su nombre fijo es **analisis-inmuebles/**.                                                                                                                               | Centraliza el almacenamiento de los archivos del sistema vinculados a los análisis inmobiliarios.         |
|  3 | Subcarpeta del proyecto             | Es la carpeta exclusiva de un proyecto concreto dentro de la carpeta principal. El documento la llama **Carpeta Id del Inmueble**. | Se crea dentro de **analisis-inmuebles/** usando como nombre el **CII** del inmueble.                                                                                    | Separa y organiza todos los archivos de cada proyecto en su propio directorio.                            |
|  4 | Nombre de la subcarpeta             | Es el nombre visible de la carpeta exclusiva del proyecto.                                                                         | Coincide con el **CII**.                                                                                                                                                 | Permite localizar rápidamente el directorio de un inmueble concreto.                                      |
|  5 | Archivo IJSON del proyecto          | Es el archivo que guarda el **IJSON completo** usado para crear el proyecto.                                                       | Se guarda dentro de la subcarpeta del proyecto y su nombre usa el **CII**. En la parte de almacenamiento se concreta como **CII.json**.                                  | Conserva la fuente estructurada original del inmueble y no se elimina en nuevas ejecuciones del análisis. |
|  6 | Archivos de análisis                | Son los archivos **Markdown** generados por cada paso analítico del proceso.                                                       | Cada archivo debe llevar como prefijo **CII_**.                                                                                                                          | Guardan los resultados individuales de cada análisis para su consulta en pantalla desde el proyecto.      |
|  7 | Archivo de análisis físico          | Es el archivo del análisis físico del inmueble.                                                                                    | **CII_analisis-fisico.md**                                                                                                                                               | Guarda el resultado del análisis físico.                                                                  |
|  8 | Archivo de análisis estratégico     | Es el archivo del análisis estratégico del inmueble.                                                                               | **CII_analisis-estrategico.md**                                                                                                                                          | Guarda el resultado del análisis estratégico.                                                             |
|  9 | Archivo de análisis financiero      | Es el archivo del análisis financiero del inmueble.                                                                                | **CII_vanalisis-financiero.md**  | Guarda el resultado del análisis financiero.                                                              |
| 10 | Archivo de análisis regulatorio     | Es el archivo del análisis regulatorio del inmueble.                                                                               | **CII_vanalisis-regulatorio.md** | Guarda el resultado del análisis regulatorio.                                                             |
| 11 | Archivo de lectura para inversor    | Es el archivo con la lectura específica para perfil inversor.                                                                      | **CII_lectura-inversor.md**                                                                                                                                              | Guarda la lectura orientada a inversor.                                                                   |
| 12 | Archivo de lectura para operador    | Es el archivo con la lectura específica para emprendedor u operador.                                                               | **CII_lectura-operador.md**                                                                                                                                              | Guarda la lectura orientada a operador.                                                                   |
| 13 | Archivo de lectura para propietario | Es el archivo con la lectura específica para propietario.                                                                          | **CII_lectura-propietario.md**                                                                                                                                           | Guarda la lectura orientada a propietario.                                                                |
| 14 | Relación entre carpeta y archivos   | Es la estructura completa de almacenamiento por proyecto.                                                                          | **analisis-inmuebles/CII/** y dentro de esa subcarpeta: **CII.json**, los **Markdown** del análisis y **CII_log.json**.                                                  | Mantiene juntos, ordenados y aislados todos los archivos de un mismo proyecto.                            |
| 16 | Regla en re-ejecución del análisis            | Norma de sustitución de archivos cuando se lanza de nuevo el análisis.                                                              | Se borran los archivos **Markdown** del directorio del proyecto, pero **no** el archivo **JSON**.                                                                        | Permite rehacer el análisis completo sin perder el IJSON original del proyecto.                           |
